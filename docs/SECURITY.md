# Política de seguridad

## Versiones soportadas

`hotframe` está en pre-alpha. No hay versiones publicadas todavía.

Cuando haya releases, soportaremos con parches de seguridad la última versión minor de cada major activa, y la minor anterior por 6 meses tras una nueva release.

## Reportar una vulnerabilidad

**No abras un issue público.** Envía un correo a **security@erplora.com** con:

1. Descripción de la vulnerabilidad.
2. Pasos para reproducir.
3. Impacto potencial.
4. Tu contacto (opcional si quieres crédito).

### SLA de respuesta

- Acuse de recibo: ≤48h laborables.
- Evaluación inicial: ≤7 días.
- Parche para vulnerabilidades críticas: ≤14 días desde confirmación.
- Disclosure coordinado: público tras parche + 30 días de grace period (o menos si se acuerda).

### Reconocimiento

Los reportes válidos se reconocen en [CHANGELOG.md](CHANGELOG.md) y en el advisory de GitHub (con permiso del reportante).
